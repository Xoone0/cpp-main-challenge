# C Branching and Switch Note 02

## 今天这一部分我在练什么

今天我继续在第三章里练判断和分支。

和前面那种“输入一个值、输出一个结果”的线性流程不同，
今天开始更明显地接触到：

- 一个问题要先分情况
- 不同情况要走不同分支
- 分支写法虽然短，但逻辑必须先想清楚

今天这一组练习让我更直接地体会到：
程序不是一直顺着往下写，
而是经常要根据条件做不同处理。

---

## 我今天最明显的感受

我今天最明显的感受是：

有些题真正难的不是代码长度，
而是“我能不能先把判断逻辑讲明白”。

比如：

- 两个数的差值怎么处理更清楚
- 三个数里谁最小
- 三个数有几个相等
- 两个数的差值是不是超出某个范围
- 一个整数是奇数还是偶数
- 一个月份应该归到哪个季节

这些题看起来都不大，
但它们都在训练一件事：

**先分情况，再写代码。**

有时候题能做出来，但代码结构和分支表达还不够清楚。

### 3. 还不够稳

今天只是继续往里推进了一部分。

## 我现在对这一章的理解

这一章真正练的，不只是 if 和 switch，
而是最基础的程序判断能力。

也就是：

- 先判断有哪些情况
- 再决定每种情况怎么处理
- 最后让代码把这些情况表达出来

这一步虽然难，
但它是后面所有程序逻辑的基础。
