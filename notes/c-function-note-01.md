# C Function Note 01

## 今天这一章在练什么

今天继续推进 C 语言第六章。

这一章开始，程序不再只是顺着写完一整段，
而是开始练：怎么把不同的处理拆成函数。

也就是说，今天真正开始接触的是：

- 函数定义
- 返回值
- 形参和实参
- 函数调用
- 数组作为参数
- `static` 在函数里的作用

和前面的分支、循环、数组相比，
函数更像是在训练我：

- 怎么拆逻辑
- 怎么复用处理
- 怎么让程序结构更清楚

---

## 我今天最明显的感受

我今天最明显的感受是：

函数表面上只是“把一段代码单独写出来”，
但真正难的地方在于：

- 这个逻辑该不该抽成函数
- 参数应该传什么
- 返回值应该是什么
- 函数和主流程怎么配合
- 数组传进去以后，函数到底改不改原数据

所以这一章不是简单的“学会写一个新格式”，
而是开始系统地组织程序结构。

---

## 我今天更清楚的地方

今天让我更清楚的是：

函数真正有价值的地方，不是形式，而是抽象。

比如今天碰到的这些题：

- 求最小值
- 求立方
- 求和
- 倒序显示数组
- 查找目标值下标
- 统计函数调用次数

它们虽然题型不同，
但本质上都在练同一个核心：

**把一段独立逻辑拆出来，让它可以被单独理解和重复使用。**

---

## 我今天暴露出来的问题

### 1. 函数能写，不代表函数边界清楚
有时候题能做出来，但函数拆分还不够干净。

### 2. 数组作为参数更抽象
和普通整数参数不同，数组参数更容易让人搞混“传进去的到底是什么”。

### 3. `static` 这种内容需要慢慢消化
代码很短，但它背后涉及的是对象寿命和状态保留，不是看一眼就会。

---

## 当前一句话总结

今天这批练习让我更明显地感受到：
第六章真正训练的，是“如何用函数把程序逻辑拆开并组织起来”。
